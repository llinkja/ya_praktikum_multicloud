Мультиоблачная модель
Плюсы:
 1) Отсутствие привязки к одному поставщику
 2) Оптимизация затрат на хранение данных в облаке
 3) Соблюдение требований региональных регуляторов
 4) Улучшение отказоустойчивости IT-инфраструктуры
 5) Доступ к передовым отраслевым технологиям
Минусы:
 1) Сложность управления и автоматизации
 2) Широкие возможности для атаки
 3) Расходы на инфраструктуру
 4) Сложность мониторинга
 5) Сложность реализации внутренних политик

Мультиоблачный подход очень гибкий и в то же время довольно хрупкий. Такая модель идеально подойдет для реализации проектов с непрогнозируемой потребностью ресурсов – например, для тестирования новых решений. Принципы работы DevOps должны способствовать интеграции сервисов в «диверсифицированных» облаках.
